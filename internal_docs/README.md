# Internal Documentation

This folder contains design docs, diagrams, and internal notes.

**This folder is gitignored** - contents are not tracked in version control.

## What goes here:

- Architecture diagrams
- Design decisions
- Meeting notes
- RFC drafts
- Internal project planning
- Brainstorming docs

## Guidelines:

- Keep a logical structure
- Use Markdown for text docs
- Use tools like Excalidraw/Mermaid for diagrams
- Reference external docs when needed

## Suggested Structure:

```
internal_docs/
  ├── architecture/
  │   ├── system_design.md
  │   └── diagrams/
  ├── decisions/
  │   ├── adr-001-choose-uv.md
  │   └── adr-002-workflow-types.md
  ├── meeting_notes/
  └── rfcs/
```
